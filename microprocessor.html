Custom 32-bit Microprocessor
with FP ALU on FPGA

A ground-up design executing IEEE-754 floating-point operations via a custom ISA, pipelined datapath, and optimized computational modules. Synthesized and deployed on a CMOD A7-35T FPGA for real-time performance analysis.

High-Level Architecture

This design employs a Load/Store structure with dual-ported BRAM enabling single-cycle instruction/data fetch. Central to the datapath is a 32-bit ALU handling FP32 operations, coordinated by an FSM-based Control Unit.

  • 32-bit Program Counter with dynamic branching and PC increment
  • Control Unit FSM issuing LOAD, EXECUTE, and WRITE signals
  • Zero flag logic for conditional BEQ operations
  • Dedicated DONE flags for multiplier/divider completion

Instruction Set Architecture (ISA)

A concise set of 16 opcodes encoded on 32 bits: from immediate FP arithmetic (FADDI, FMULI) to memory accesses (LDA, STORA) and program flow (JMP, BEQ). All decoding is parallel to minimize cycle time.

Functional Verification

We preloaded our BRAM with a representative instruction sequence covering FP add/sub, mul/div, logical ops, and branches. Waveform analysis confirmed correct result propagation, PC updates, and zero-flag behavior across all cycles.

Live FPGA Debug with ILA

Using Xilinx ILA cores, we captured internal signals (clk_ALU, op_code, LoadData, MUL_DONE, DIV_DONE) live on a CMODA7-35T. This real-time visibility was crucial for timing closure and fine-tuning the pipeline.
